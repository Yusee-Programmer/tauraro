# Benchmark Results Conflict Resolution

## Problem
The `benchmarks/results.md` file is auto-generated by CI on every commit to the master branch. This can cause merge conflicts when:
- Your fork's master branch has a different version of results.md
- The upstream organization repo has been updated with new benchmark results
- You're creating a PR from your fork to the organization repo

## Solution

### Permanent Configuration
We've configured three layers of conflict resolution:

1. **`.gitattributes` - Merge Strategy**
   ```
   benchmarks/results.md merge=ours
   ```
   This tells git to use your version when conflicts occur.

2. **`.github/workflows/auto-resolve-conflicts.yml`**
   A GitHub Action that can automatically rebase PRs with conflicts using `rebase -X ours`.

3. **Helper Script**
   A shell script to easily resolve conflicts locally.

### How to Resolve Conflicts

#### Option 1: Use the Helper Script (Recommended)
```bash
bash scripts/resolve-benchmark-conflicts.sh
```

Or specify a different remote branch:
```bash
bash scripts/resolve-benchmark-conflicts.sh upstream/master
```

#### Option 2: Manual Resolution
When you encounter a conflict during rebase:
```bash
git rebase -X ours origin/master
# or
git rebase -X ours upstream/master
```

The `-X ours` strategy automatically resolves conflicts by keeping your version of conflicting files.

#### Option 3: During Pull/Merge
```bash
git pull -X ours upstream master
```

### Why `ours` Strategy Works
- `benchmarks/results.md` is **auto-generated** - conflicts don't represent real data loss
- Both versions are snapshots of benchmark runs from different times
- Taking the current branch version is safe since we can always re-run benchmarks
- No manual conflict markers need resolution

### Example Workflow
1. You have changes in your fork on `master`
2. You notice PR conflicts with upstream/master
3. Run: `bash scripts/resolve-benchmark-conflicts.sh upstream/master`
4. Review the rebase: `git log -5`
5. Force push: `git push origin master --force-with-lease`

### Prevention
The GitHub Action in `.github/workflows/auto-resolve-conflicts.yml` can be triggered on PRs to automatically resolve these conflicts, though it requires proper permissions and configuration in the org repo.
